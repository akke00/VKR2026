# Модификация метода SemDeDup на основе локально-адаптивных порогов

Программная реализация и экспериментальное сравнение методов семантической
дедупликации текстовых корпусов. Проект сопровождает магистерскую диссертацию,
посвящённую модификациям метода SemDeDup с использованием локально-адаптивных
порогов семантической близости.

## Описание

Базовый метод SemDeDup кластеризует векторные представления документов и удаляет
внутри каждого кластера пары, косинусная близость которых превышает единый
глобальный порог. Ключевое ограничение метода — фиксированный порог не учитывает
неоднородность эмбеддингового пространства. Проект реализует базовый метод и шесть
адаптивных модификаций, вычисляющих порог локально, и сравнивает их на корпусе
Quora Question Pairs.

## Реализованные методы

| Метод | Функция | Уровень адаптации | Принцип |
|-------|---------|-------------------|---------|
| SemDeDup (базовый) | `semdedup` | — | единый глобальный порог |
| Density-Adaptive Threshold | `semdedup_density_adaptive` | кластер | порог зависит от плотности кластера |
| k-NN Local Threshold | `semdedup_knn_local` | документ | квантиль близостей к k соседям |
| Percentile-based Cluster Threshold | `semdedup_percentile_cluster` | кластер | перцентиль внутрикластерных близостей |
| Cluster Mean+Std | `semdedup_cluster_mean_std` | кластер | среднее + α·std близостей |
| Otsu Adaptive Threshold | `semdedup_oat` | кластер | бинаризация распределения методом Оцу |
| Two-Pass OAT | `semdedup_two_pass` | кластер + группа | два прохода: поиск кандидатов + верификация |

## Структура проекта

```
semdedup.py              Базовый метод SemDeDup; общие утилиты (k-means,
                         структура результата SemDeDupResult)
semdedup_adaptive.py     DAT, kNN-LT, Percentile-based Cluster Threshold
semdedup_mean_std.py     Cluster Mean+Std
semdedup_otsu.py         OAT и Two-Pass OAT
embeddings_sbert.py      Получение SBERT-эмбеддингов (all-MiniLM-L6-v2)
data_qqp.py              Загрузка корпуса Quora Question Pairs
evaluation_pairs.py      Попарные метрики качества (precision, recall, F1)
grid_search_qqp.py       Подбор гиперпараметров (точка входа)
benchmark_qqp_final.py   Финальный бенчмарк по размерам корпуса (точка входа)
```

Граф зависимостей: обе точки входа (`grid_search_qqp.py`, `benchmark_qqp_final.py`)
используют `data_qqp`, `embeddings_sbert`, `evaluation_pairs` и все модули методов;
модули методов и `evaluation_pairs` импортируют общие компоненты из `semdedup`.


## Запуск

Эксперимент состоит из двух шагов. Сначала подбираются гиперпараметры на отдельной
выборке, затем проводится итоговое сравнение методов.

```bash
# Шаг 1. Подбор гиперпараметров (выборка N=2000, seed=0)
python grid_search_qqp.py

# Шаг 2. Итоговый бенчмарк по размерам корпуса (seeds 1, 2, 3)
python benchmark_qqp_final.py
```

Первый шаг создаёт `results_full/best_params_qqp.json` с лучшими параметрами каждого
метода. Второй шаг читает этот файл; если он отсутствует, подбор запускается
автоматически.

## Методика эксперимента

Подбор параметров и итоговая оценка проводятся на **непересекающихся выборках**,
что исключает утечку данных: подбор использует seed=0, бенчмарк — seeds 1, 2, 3.

Бенчмарк прогоняется на размерах корпуса N ∈ {50, 100, 500, 1000, 2000, 5000,
10000}. Каждый размер прогоняется трижды (seeds 1, 2, 3), результат приводится как
среднее ± стандартное отклонение. Размеры N = 50 и N = 100 помечаются как
иллюстративные: при них мало эталонных пар, и результаты статистически ненадёжны.

Качество оценивается **попарно**: для пары документов проверяется, корректно ли она
определена как дубликат, относительно эталонной разметки QQP. Метрики — точность,
полнота, F1-мера, а также число ложно удалённых уникальных документов
(необратимая потеря информации).

## Результаты

Файлы, создаваемые при запуске:

```
results_full/best_params_qqp.json     лучшие гиперпараметры каждого метода
results_full/grid_search_qqp.csv       все конфигурации подбора
results_full/qqp_benchmark_raw.csv     результаты всех прогонов (по seed)
results_full/qqp_benchmark_final.csv   агрегированные результаты (mean ± std)
figures_full/                          графики масштабирования
```

Основные выводы исследования: метод k-NN Local Threshold превосходит базовый
SemDeDup по F1-мере на всех размерах корпуса (прирост 5–8 процентных пунктов), а
метод Two-Pass OAT обеспечивает наименьшее число ложно удалённых уникальных
документов. Методы кластерной адаптации (DAT, PCT, Mean+Std, OAT) на этом корпусе
не дают существенного преимущества над базовым методом с подобранным порогом.

## Источник данных

Корпус Quora Question Pairs загружается автоматически из HuggingFace Hub при первом
запуске. Используется датасет `AlekseyKorshuk/quora-question-pairs` (с запасным
источником при недоступности). Загруженные данные кэшируются в памяти на время
работы скрипта.

## Использование методов отдельно

Все методы принимают на вход массив L2-нормализованных эмбеддингов формы (N, d) и
возвращают единую структуру `SemDeDupResult` с индексами сохранённых и удалённых
документов, списком пар дубликатов

```python
import numpy as np
from embeddings_sbert import TextEmbedder, SBERTConfig
from semdedup_adaptive import semdedup_knn_local
from evaluation_pairs import evaluate_pairs

texts = ["первый вопрос", "почти такой же вопрос", "совсем другой текст"]
embedder = TextEmbedder(SBERTConfig(model_name="all-MiniLM-L6-v2"))
emb = embedder.fit_transform(texts)

result = semdedup_knn_local(emb, k=5, quantile=0.80, min_threshold=0.70)
clean_texts = [texts[i] for i in result.keep_indices]
```
